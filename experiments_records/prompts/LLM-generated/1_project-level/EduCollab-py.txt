You are a senior Python engineer. Build a minimal but WORKING Python 3.11+ Flask web application using an MVC-style architecture. Implement ONLY the functional requirements below and keep everything else minimal and runnable.

Language/Framework
- Python 3.11+
- Flask
- SQLite: use sqlite3 (standard library) OR SQLAlchemy (choose ONE and use it consistently; prefer sqlite3 for minimal deps)
- Sessions: Flask session (cookie-based default) or Flask-Session for server-side; choose ONE and use consistently
- Server-rendered templates: Jinja2 (Flask default)
- MUST provide BOTH HTML pages and JSON REST API under /api
- HTML and /api MUST use the SAME database and the SAME session/auth logic

Architecture requirement (MVC, function-only; no fixed file paths)
- Model/Repository: DB access only
- Controller: Flask route handlers calling repos
- View: Jinja templates
- Keep dependencies minimal
- Provide a minimal app entry (app.py) or small package; must be runnable

Scope (v0) Objects/modules
- User v0
- Course v0
- Membership v0 (roles: admin/teacher/student/assistant/senior-assistant)
- Post + Comment + Search (any logged-in user can operate within course context)
- Assignment + Submission + Upload
  - teacher/assistant/senior-assistant can manage Assignment + Upload
  - student can submit Submission (and view/update own)
- Quiz + Question + Question Bank
  - teacher/assistant/senior-assistant can manage question bank + quiz
  - student can take quiz only

Data model (minimum)
- User(user_id, username, password_hash, display_name, created_at)
- Course(course_id, title, description, created_by, created_at)
- Membership(membership_id, course_id, user_id, role_in_course, created_at)
  - role_in_course ∈ {admin, teacher, student, assistant, senior-assistant}
- Post(post_id, course_id, author_id, title, body, created_at, updated_at)
- Comment(comment_id, post_id, course_id, author_id, body, created_at, updated_at)
- Assignment(assignment_id, course_id, created_by, title, description, due_at, created_at, updated_at)
- Submission(submission_id, assignment_id, course_id, student_id, content_text, attachment_upload_id nullable, created_at, updated_at)
- Upload(upload_id, course_id, uploaded_by, original_name, storage_path, created_at)
- Question(question_id, course_id, created_by, qtype, prompt, options_json nullable, answer_json nullable, created_at, updated_at)
- Quiz(quiz_id, course_id, created_by, title, description, open_at, due_at, created_at, updated_at)
- QuizQuestion(quiz_id, question_id, points, position)
- QuizAttempt(attempt_id, quiz_id, course_id, student_id, started_at, submitted_at nullable)
- QuizAnswer(answer_id, attempt_id, question_id, answer_json, created_at)

Role rules (must implement)
- Post/Comment/Search: any logged-in user can CRUD within a course context.
- Assignment: teacher/assistant/senior-assistant can create/update/delete; any logged-in user can view list/detail.
- Submission: student can create/update/view own; teacher/assistant/senior-assistant can view all submissions for an assignment.
- Upload: teacher/assistant/senior-assistant can upload/delete; any logged-in user can view list and download.
- Question bank: teacher/assistant/senior-assistant CRUD questions.
- Quiz: teacher/assistant/senior-assistant CRUD quizzes and configure questions; student can only take quiz (attempt + submit) and view own attempts (optional).
- Membership: teacher/admin can add/remove members and set role_in_course; any logged-in user can view course members list.
