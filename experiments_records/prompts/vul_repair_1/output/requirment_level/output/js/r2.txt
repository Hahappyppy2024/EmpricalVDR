2. Per-vulnerability summary

**VULN-001**

* files changed

  * `src/routes/web.js`
  * `src/routes/api.js`
* key checks added

  * Added `loadCourseMembership` + `requireCourseMember` to `GET /courses/:course_id/members`
  * Added `loadCourseMembership` + `requireCourseMember` to `GET /api/courses/:course_id/members`
* why the vulnerability is mitigated

  * Roster reads now require that the caller already has a membership in the target course before the controller can return membership data, which closes the arbitrary `course_id` read path.

**VULN-002**

* files changed

  * `app.js`
  * `src/middleware/auth.js`
  * `src/routes/web.js`
  * `src/views/course_member_new.ejs`
  * `src/views/course_members_list.ejs`
* key checks added

  * Added per-session CSRF token generation in `app.js`
  * Exposed `csrfToken` to server-rendered views
  * Added reusable `requireValidPageCsrf` middleware
  * Applied CSRF validation to membership-management POST routes
  * Added hidden `_csrf` fields to add/update/remove membership forms
* why the vulnerability is mitigated

  * State-changing membership form submissions now require a token that matches the current session, so a third-party site cannot forge those POST requests using only the victim’s session cookie.
