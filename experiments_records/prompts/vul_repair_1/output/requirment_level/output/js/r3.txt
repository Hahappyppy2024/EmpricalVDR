## 2) Per-vulnerability summary

### VULN-001 (Post IDOR: edit/delete others’ posts)

* **Files changed**

  * `src/controllers/postController.js`
  * `src/repositories/postRepository.js`
* **Key checks added**

  * Controller now enforces: caller must be **post author** OR **course staff** (`teacher/admin/assistant/senior-assistant`) for:

    * web edit page (`GET .../edit`)
    * web update (`POST /courses/:course_id/posts/:post_id`)
    * web delete (`POST /courses/:course_id/posts/:post_id/delete`)
    * API update (`PUT /api/courses/:course_id/posts/:post_id`)
    * API delete (`DELETE /api/courses/:course_id/posts/:post_id`)
  * Repository adds author-scoped write APIs:

    * `updatePostAsAuthor(postId, authorId, ...)` uses `WHERE post_id = ? AND author_id = ?`
    * `deletePostAsAuthor(postId, authorId)` deletes only if `author_id` matches (and only then deletes the post’s comments)
* **Why mitigated**

  * Possessing a `post_id` is no longer sufficient: non-staff users can only mutate rows they authored, and staff-only moderation remains possible.

### VULN-002 (Comment IDOR: edit/delete others’ comments)

* **Files changed**

  * `src/controllers/commentController.js`
  * `src/repositories/commentRepository.js`
* **Key checks added**

  * Controller now enforces: caller must be **comment author** OR **course staff** for:

    * web update/delete routes
    * API update/delete routes
  * Repository adds author-scoped write APIs:

    * `updateCommentAsAuthor(commentId, authorId, ...)` uses `WHERE comment_id = ? AND author_id = ?`
    * `deleteCommentAsAuthor(commentId, authorId)` uses `WHERE comment_id = ? AND author_id = ?`
* **Why mitigated**

  * A course member cannot tamper with another member’s comment by `comment_id` alone; only authors (or staff moderators) can update/delete.
