2. Per-vulnerability summary (group by vulnerability_id)

VULN-001

* files changed

  * `app/core/Auth.php`
  * `app/controllers/PostController.php`
* key checks added

  * Added `can_moderate_course_content(...)` to narrow non-owner post moderation to `teacher` and `admin` only.
  * Kept `can_manage_course_content(...)` author-aware, but changed its non-owner path to use the narrower moderator check instead of the broad staff-role helper.
  * Changed post update/delete controller branching so unrestricted repository writes are used only for true moderators, not for `assistant` or `senior-assistant`.
* regression restored

  * No route, response field, or role-name contract changed.
  * Legitimate author CRUD remains intact because authors still pass `can_manage_course_content(...)` and still use the owner-scoped repository path.
* why the vulnerability is mitigated

  * `assistant` and `senior-assistant` no longer inherit blanket authority to mutate another author’s post by supplying `post_id`.

VULN-002

* files changed

  * `app/core/Auth.php`
  * `app/controllers/CommentController.php`
* key checks added

  * Reused the new `can_moderate_course_content(...)` helper so only `teacher` and `admin` can bypass comment ownership.
  * Changed comment update/delete controller branching so unrestricted repository writes are no longer available to `assistant` or `senior-assistant`.
* regression restored

  * Comment author CRUD remains intact because comment authors still pass the ownership check and continue to use the owner-scoped repository path.
* why the vulnerability is mitigated

  * `assistant` and `senior-assistant` can no longer update or delete another author’s comment by supplying `comment_id`.
