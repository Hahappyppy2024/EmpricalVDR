2. Per-vulnerability summary (group by vulnerability_id):

**VULN-001**

* files changed

  * `app/controllers/PostController.php`
* key checks added

  * Kept the narrowed author-or-true-moderator authorization path already present in `Auth.php`.
  * Restored controller compatibility with the repository’s real constructor style used by `public/index.php`.
  * Restored the expected route methods `showCreateForm`, `createHtml`, and `createApi`.
  * Restored repository call shape to `PostRepository::create(array $data)`.
* regression restored

  * Fixed the startup/login regression caused by requiring nonexistent `app/core/Controller.php`.
  * Fixed the constructor mismatch that broke controller instantiation during bootstrap.
  * Fixed the wrong view path `courses/posts/create` back to existing `courses/posts/new`.
* why the vulnerability is mitigated (brief)

  * The controller now runs in the intended app architecture again, while the server-side ownership boundary remains narrowed so `assistant` and `senior-assistant` no longer get blanket post-moderation power.

**VULN-002**

* files changed

  * `app/controllers/CommentController.php`
* key checks added

  * Kept the narrowed author-or-true-moderator authorization path already present in `Auth.php`.
  * Restored controller compatibility with the repository’s real constructor style used by `public/index.php`.
  * Restored the expected route methods `listHtml`, `createHtml`, and `createApi`.
  * Restored repository call shape to `CommentRepository::create(array $data)`.
  * Preserved `can_manage` and `can_manage_post` flags when re-rendering the post page after HTML comment-creation validation failures.
* regression restored

  * Fixed the same startup/login regression chain caused by the incompatible base-controller rewrite.
  * Fixed missing methods expected by `public/index.php` route dispatch.
* why the vulnerability is mitigated (brief)

  * The comment controller now matches the current repository contract again, and the server still enforces author ownership unless the caller is a real course moderator (`teacher` or `admin`).
