2. Per-vulnerability summary

**VULN-001**

* files changed

  * `src/routes/api.js`
  * `src/routes/web.js`
  * `src/controllers/questionController.js`

* key checks added

  * Restored question list/detail read access from `requireCourseStaff` back to `requireCourseMember`.
  * Added role-aware field projection in `questionController`.
  * Staff still receive full question records, while non-staff members receive `listByCourseSafe()` / `findByIdSafe()` results without `answer_json`.

* regression restored

  * The requirement-stage student question detail flow now returns 200 again instead of the over-restrictive 403 introduced by the previous repair.

* why the vulnerability is mitigated

  * Students can still use the requirement-stage question read flow, but sensitive answer data is no longer disclosed through question list/detail responses.
  * Staff-only answer visibility is preserved without breaking the member-facing flow.

**VULN-002**

* files changed

  * `src/repositories/quizRepository.js`

* key checks added

  * Hardened `saveAnswer()` so the write layer independently verifies that the target `question_id` belongs to the attempt’s quiz before persisting.
  * This supplements the controller-level `isQuestionInQuiz()` check added in the previous repair.

* regression restored

  * None.

* why the vulnerability is mitigated

  * Arbitrary foreign question IDs are now blocked both at the controller layer and at the repository write boundary.
  * Even if another caller reaches `saveAnswer()` later, invalid answer injection is still rejected.
