The root cause is now clearly identified.

The problem is not with the 9 security tests themselves. The real issue is that the application returns **500 on the very first `GET /login`** after startup, so the fixture never sees a healthy response and therefore all tests fail with:

`RuntimeError: Flask server did not start`

The actual error is:

```python
werkzeug.routing.exceptions.BuildError:
Could not build url for endpoint 'my_memberships_page'. Did you mean 'my_memberships' instead?
```

That means this line in `templates/base.html` is wrong:

```html
<a class="btn" href="{{ url_for('my_memberships_page') }}">My Memberships</a>
```

It should be changed to:

```html
<a class="btn" href="{{ url_for('my_memberships') }}">My Memberships</a>
```

I also checked the related endpoint names, and the result is:

* `my_memberships_page` ❌ does not exist
* `my_memberships` ✅ exists
* `my_submissions_page` ✅ exists
* `my_quiz_attempts_page` ✅ exists
* `me_page` ✅ exists

So right now, you only need to fix **this one line**.

The corrected `templates/base.html` should be:

```html

<html><head><meta charset="utf-8"><title>{{ title or "EduCollab v0" }}</title>
```
