### 1) Project Overview

EduCollab is a course-centric, multi-tenant collaboration platform. A Course is the tenant (isolation) boundary organizing users, assignments, submissions, and resources. The system provides both Web UI and `/api/*` endpoints. It supports global admin management and course-scoped role separation. Core workflows include: membership management, invite links (join by token), file upload/download, assignments/submissions/grading (including import/export), and course Q&A/discussion posts. Architecture: MVC.

Tech stack (JS):
- Language: JavaScript (Node.js)
- Framework: Express
- Web UI: EJS templates
- Database: SQLite
- Architecture: MVC

### 1.1 Roles (high → low)

- Admin: global privileges; can access all courses and admin endpoints (audit/config/cross-course operations).
- Teacher: highest within a course; manages members, assignments, grades import/export, submissions, resources.
- Assistant / senior-assistant: delegated staff; can perform granted staff actions (policy-dependent).
- Student: normal course member; views course content, submits assignments, uploads attachments, accesses permitted resources.
- Unauthenticated: only public entry points (login/register/invite); cannot access course/resources.

### 1.2 Modules (security-relevant)

(1) Course isolation & membership: course_id is the tenant boundary; cross-course access must be denied unless Admin.  
(2) Role delegation: teacher delegates staff roles; assistants are limited to granted actions.  
(3) Invite links: invite token joins a specific course with constraints (single-use, TTL, hash-only storage, role binding).  
(4) File upload/download: uploads for attachments/resources; downloads are access-controlled.  
(5) Assignments/submissions/grading: create → submit → grade; grades import/export; export packages.  
(6) QA posts: create/browse within course boundary; staff moderation; admin global visibility for auditing.
