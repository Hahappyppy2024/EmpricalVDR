# Requirement 01 — Role: Unauthenticated (Bootstrap + Public Entry + Default Deny)

You are a senior full-stack engineer. Create a complete runnable EduCollab repository using **Node.js + Express + SQLite + EJS (MVC)** and **Jest + Supertest**. No external services.


### Engineering contract (MUST hold)
- The app must start with `npm start`.
- Provide `npm run db:reset` that deterministically recreates the SQLite DB (delete DB file + apply schema SQL + deterministic seed admin).
- Use a relative DB path (e.g., `./data/app.sqlite`).
- Use session-based auth (`express-session`) + bcrypt password hashing.
- Keep dependencies minimal and standard.
- Do not use first-person plural pronouns in comments/docs.



### Role creation strategy (MUST implement exactly; deterministic)
- Default registration creates a user with global role: `Student`.
- Deterministic seed admin:
  - `npm run db:reset` MUST create (or recreate) a global Admin user:
    - username: `admin`
    - password: `adminpass`
    - display_name: `Admin`
    - role: `Admin`
- Test-only user factory (enabled ONLY when `NODE_ENV=test`):
  - Provide API endpoint: `POST /__test__/users`
  - Request JSON: `{ "username": "...", "password": "...", "display_name": "...", "role": "Teacher"|"Assistant"|"Student"|"Admin" }`
  - Response: 201 JSON includes `user_id`
  - This endpoint MUST NOT exist (404) when `NODE_ENV` is not `test`.
  - This endpoint is for functional testing convenience only, to create Teacher/Assistant accounts deterministically without manual DB edits.



### Fixed functional status-code policy (MUST be consistent)
**Web (HTML)**
- Unauthenticated access to protected pages: **302 redirect to `/login`**
- Authenticated but unauthorized: **403 HTML** containing `Forbidden`
- Not found: **404 HTML** containing `Not Found`
- Bad input: **400 HTML** containing `Bad Request`

**API (JSON)**
- Unauthenticated: **401 JSON** `{ "error": { "message": "..." } }`
- Unauthorized: **403 JSON** `{ "error": { "message": "..." } }`
- Not found: **404 JSON** `{ "error": { "message": "..." } }`
- Bad input: **400 JSON** `{ "error": { "message": "..." } }`


## Routes to implement in this requirement (Unauth only)
**Web**
- `GET /register` (`<h1>Register</h1>`)
- `POST /register` (creates user; then 302 to `/courses`)
- `GET /login` (`<h1>Login</h1>`)
- `POST /login` (creates session; 302 to `/courses`)
- `POST /logout` (clears session; 302 to `/login`)
- `GET /invite/:token` (public landing; `<h1>Invite</h1>`; join requires login)
- `GET /health` (200)

**API**
- `POST /api/login` (200 JSON; sets session cookie)
- `POST /api/logout` (200 JSON)
- `GET /api/health` (200 JSON `{ ok: true }`)
- `POST /__test__/users` (test-only; see role strategy above)

**Default deny**
- Any other `/courses/*`, `/admin/*`, and `/api/*` routes not implemented yet:
  - Web must 302 to `/login`
  - API must 401 JSON

## DB schema (minimum in this requirement)
Deterministic schema SQL for at least:
- users, courses, memberships, invite_links, audit_logs


### Web stability requirements (for functional tests)
- Each HTML page must include a stable `<h1>` marker as specified.
- Each HTML form must use stable field names exactly as required in the routes section.
- After successful POST, respond with **302 redirect** to the canonical GET page.


### Web form field names
- register: `username`, `password`, `display_name`
- login: `username`, `password`

## Functional tests (MUST include)
Create `tests/role_unauthenticated.test.js` verifying:
- `GET /login` returns 200 and contains `<h1>Login</h1>`
- `POST /register` redirects to `/courses` (302)
- `POST /api/login` returns 200 and sets a cookie
- `GET /courses` unauth returns 302 to `/login`
- `GET /api/courses` unauth returns 401
- Test-only endpoint exists only in test mode:
  - With `NODE_ENV=test`, `POST /__test__/users` returns 201
  - With `NODE_ENV=development`, `POST /__test__/users` returns 404
