### 4.3 RootCause

RootCause:

* `GET /api/courses/:course_id/posts` does not enforce **course membership** for the requesting user (missing course-isolation check), so a user from course2 can read course1 posts (IDOR).

* Enforcement is missing at the correct boundary: it must happen **before** the posts query/response, ideally as a reusable middleware applied to all `/api/courses/:course_id/*` resources (or at least in the posts controller).
