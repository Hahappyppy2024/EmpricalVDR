4.3 RootCause
- The endpoint `GET /api/courses/:course_id/posts` lacks an authorization check to verify that the requesting user is a member of the specified course (`course_id`).
- The application relies solely on authentication (user is logged in) but fails to enforce resource-level authorization, allowing authenticated users to access data from any course (IDOR).

4.4 ActionablePlan
- Target File: routes/courses.js (or relevant router file)
  Target: GET /api/courses/:course_id/posts route
  Change: Add middleware or a conditional check at the beginning of the handler to verify that `req.user.id` is associated with `req.params.course_id` in the course members table. Return 403 Forbidden if the association does not exist.

- Target File: middleware/auth.js (or similar)
  Target: Authorization logic
  Change: Implement or utilize a middleware function specifically designed to check course membership before accessing course-scoped resources.
