4.3 RootCause
- The endpoint POST /api/courses/:course_id/assignments fails to verify the authenticated user's role within the specific course context.
- It allows any authenticated user (including those with the 'student' role) to create assignments, violating the Role-based access control property requiring teacher/admin privileges.

4.4 ActionablePlan
- Target File: routes/assignments.js (or relevant route file)
  Target: POST /api/courses/:course_id/assignments route handler
  Change: Add authorization logic to retrieve the user's role for the specified course_id. If the role is 'student', return 403 Forbidden.

- Target File: middleware/auth.js (or controller)
  Target: Assignment creation controller function
  Change: Ensure the controller checks req.user.role or queries the course_members table to enforce that only non-student roles can proceed.

4.5 FileToActionMap
- routes/assignments.js → Add role-based authorization check to the POST endpoint.
- controllers/assignmentController.js → Enforce role check logic before creating the assignment.
