4.3 RootCause
- The GET /api/courses endpoint is publicly accessible and lacks authentication enforcement.
- There is no authentication middleware mounted for the /api/* routes or specifically for the /api/courses route.
- The route handler processes the request without verifying session validity or user identity, resulting in a successful response (likely HTTP 200) instead of a rejection (HTTP 401/302).

4.4 ActionablePlan
- Target File: routes/courses.js (or equivalent route file)
  Target: GET /api/courses route definition
  Change: Add authentication middleware (e.g., `authMiddleware`) to the route or router chain to intercept unauthenticated requests.
- Target File: routes/courses.js (or equivalent route file)
  Target: Route Handler
  Change: Ensure the handler is only reached after successful authentication, or explicitly check `req.user` / `req.session.user` and return 401 if missing.
